# Composite (Компоновщик)

Компоновщик — это структурный паттерн проектирования, который позволяет сгруппировать объекты в древовидную структуру, а затем работать с этой структурой так же, как с отдельным объектом.

## Проблема
Необходимо реализовать древовидную структуру (например, файловую систему), в которой есть конечные объекты (файлы) и контейнеры (папки). Клиентский код не должен отличать файл от папки — он должен уметь одинаково вызывать методы (например, отобразить информацию или посчитать размер) для любого элемента системы.

## Решение
Создать общий интерфейс `FileSystemNode`, который определяет общее поведение для всех элементов. И файл (`File`), и папка (`Folder`) реализуют этот интерфейс. Папка дополнительно хранит список дочерних элементов и делегирует им вызовы методов, рекурсивно обходя дерево.

## Особенности реализации на C++
* **Древовидная структура:** Папка (`Folder`) содержит вектор указателей на абстрактный класс `FileSystemNode`. Это позволяет вкладывать папки друг в друга бесконечно.
* **Рекурсия:** Методы `display()` и `getSize()` в классе `Folder` вызывают такие же методы у своих дочерних элементов, что обеспечивает естественный обход дерева.
* **Безопасные методы в базовом классе:** В интерфейсе `FileSystemNode` методы `add()` и `remove()` сделаны пустыми (виртуальными). Это позволяет клиенту не проверять тип объекта перед вызовом: если вызвать `add()` у файла, просто ничего не произойдет, и программа не упадет.
* **Освобождение памяти:** В деструкторе `Folder` происходит обход вектора и удаление всех дочерних элементов. Достаточно удалить корневую папку, чтобы очистить всю память, выделенную под дерево.

## Плюсы
* Позволяет клиенту единообразно обрабатывать как отдельные объекты, так и их композиции.
* Облегчает добавление новых видов объектов (достаточно создать новый класс, наследующий `FileSystemNode`).
* Упрощает архитектуру клиентского кода (нет кучи проверок `if (isFolder) ... else ...`).

## Минусы
* Затрудняет ограничение типов объектов, которые можно добавлять в контейнер (например, нельзя легко запретить добавлять папку внутрь файла только средствами базового интерфейса).
