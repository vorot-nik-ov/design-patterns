# Flyweight (Легковес)

Легковес — это структурный паттерн проектирования, который позволяет разделять общее состояние множества однотипных объектов в одном месте, чтобы не хранить одинаковую информацию в каждом объекте по отдельности.

## Проблема
В приложении (например, текстовом редакторе или таблице) могут создаваться тысячи объектов (ячеек). Многие из них имеют одинаковые свойства (одинаковый шрифт, размер и текст). Если хранить эти свойства внутри каждого объекта, это приведет к огромному неоправданному расходу оперативной памяти.

## Решение
Разделить состояние объекта на две части:
1. **Внутреннее состояние (intrinsic):** Данные, которые не меняются в зависимости от контекста (шрифт, размер, текст). Они выносятся в общий класс (`Cell`) и кэшируются в специальной Фабрике (`CellFactory`).
2. **Внешнее состояние (extrinsic):** Данные, уникальные для каждого контекста (координаты строки и столбца). Они не хранятся в самом легковесе, а передаются в его методы извне (хранятся в структуре `CellPosition` в классе `Table`).

## Особенности реализации на C++
* **Пул объектов:** Класс `CellFactory` использует `unordered_map` (хэш-таблицу) в качестве пула. Ключом выступает уникальная строка, сгенерированная из свойств ячейки.
* **Структура для внешнего состояния:** В классе `Table` создана структура `CellPosition`, которая связывает конкретный легковесный объект `Cell*` с его уникальными координатами на экране. Это правильный подход для работы с внешним состоянием.
* **Очистка памяти:** В деструкторе `~CellFactory()` происходит корректный обход пула и удаление всех созданных ячеек. Достаточно удалить фабрику, чтобы очистить всю занятую легковесами память.

## Плюсы
* Существенная экономия оперативной памяти при большом количестве одинаковых объектов.
* Ускорение создания объектов (если объект уже есть в пуле, он возвращается мгновенно без вызова оператора `new`).

## Минусы
* Усложняет код из-за необходимости разделять состояние на внутреннее и внешнее.
* Затраты процессорного времени на поиск нужного объекта в хэш-таблице (пуле).
* Читаемость кода снижается, так как логика работы с объектом размазывается по классам Фабрики и Контейнера.
