# Facade (Фасад)

Фасад — это структурный паттерн проектирования, который предоставляет простой интерфейс к сложной системе классов, библиотеке или фреймворку.

## Проблема
Ваша система состоит из множества сложных подсистем с запутанными взаимосвязями (пользователи, каталог фильмов, платежная система, стриминг, уведомления). Клиенту приходится понимать логику работы каждой подсистемы и в правильном порядке вызывать десятки методов, чтобы выполнить простое действие (посмотреть фильм).

## Решение
Создать класс-фасад (`CinemaFacade`), который объединяет в себе все подсистемы. Фасад принимает на себя всю рутину по координации подсистем и предоставляет клиенту один простой метод (`watchMovie`), который делает всю работу за него.

## Особенности реализации на C++
* **Композиция объектов:** В данном примере подсистемы создаются внутри Фасада как обычные объекты (по значению, а не через указатели). Это избавляет от необходимости писать деструктор и управлять памятью для подсистем, так как их время жизни совпадает с временем жизни Фасада.
* **Делегирование:** Фасад не реализует бизнес-логику подсистем сам. Он просто перенаправляет вызовы нужным объектам (`user.checkSubscription()`, `catalog.findMovie()`).
* **Управление потоком:** Фасад скрывает за собой не только вызовы методов, но и логику принятия решений (например, "если качество 4K, то списать деньги через платежную систему").

## Плюсы
* Изолирует клиентов от сложности подсистем.
* Упрощает использование сложной системы до пары простых методов.
* Снижает связность (coupling) между клиентским кодом и подсистемами.

## Минусы
* Риск создания объекта "Бога" (God Object), если Фасад начнет выполнять слишком много логики и превратиться в огромный нечитаемый класс.
* Фасад может ограничить доступ к функционалу подсистем, который может понадобиться некоторым клиентам (но это решается предоставлением прямого доступа к подсистемам рядом с Фасадом).
