# Prototype (Прототип)

Прототип — это порождающий паттерн проектирования, который позволяет копировать объекты, не вдаваясь в подробности их реализации. Он позволяет клонировать уже существующие объекты (прототипы), не завязываясь на их конкретные классы.

## Проблема
Создание объекта с множеством полей (например, мебели: материал, цвет, габариты, цена) требует инициализации всех этих параметров. Если нужно создать десять похожих стульев, отличающихся только цветом, дублировать код конструктора неудобно и ресурсоемко.

## Решение
Завести интерфейс с методом `clone()`. Базовый класс `Furniture` возвращает указатель на свой копию, а дочерние классы (Table, Chair, Sofa) реализуют этот метод, создавая копию самих себя с помощью конструктора копирования.

## Особенности реализации на C++
* **Умные указатели при клонировании:** метод `clone()` возвращает `std::unique_ptr<Furniture>`. Это гарантирует, что клиент, получивший копию, станет ее единственным владельцем, и не будет утечек памяти.
* **Использование конструктора копирования:** внутри `clone()` используется конструкция `make_unique<Table>(*this)`. Она вызывает стандартный конструктор копирования, который автоматически копирует все поля базового класса.
* **Изменение клона:** получив копию, мы можем изменить у нее только нужные поля (например, `setColor`), не трогая остальные.

## Плюсы
* Позволяет клонировать объекты без привязки к их конкретным классам (клиент работает только с абстрактным интерфейсом `Furniture`).
* Снижает нагрузку на инициализацию новых объектов, если она тяжелая.
* Упрощает добавление новых классов в программу (достаточно реализовать метод `clone()`).

## Минусы
* Сложно клонировать объекты, которые имеют ссылки на другие объекты (циклические зависимости).
* Требует реализации метода клонирования у каждого класса-наследника, что может быть утомительно при очень глубокой иерархии.
