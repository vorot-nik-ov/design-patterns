# Command (Команда)

Команда — это поведенческий паттерн проектирования, который превращает запросы в объекты, позволяя передавать их как аргументы при вызове методов, ставить в очередь, логировать или поддерживать отмену операций.

## Проблема
Вам нужно управлять устройствами (включить, использовать, выключить), но вы не хотите жестко привязывать инициатора (например, кнопку на пульте) к конкретному получателю (принтеру). Кроме того, может потребоваться объединить несколько операций в одну (например, "Запустить презентацию", что включает проектор и колонки).

## Решение
Обернуть каждый запрос в отдельный объект (команду). Объект команды реализует общий интерфейс `ICommand` и хранит ссылку на объект-получатель. Инициатор (`Button`) работает только с интерфейсом `ICommand` и не знает, какое именно устройство он управляет.

## Особенности реализации на C++
* **Макрокоманда (Macro Command):** Класс `MacroCommand` реализует тот же интерфейс `ICommand` и содержит вектор других команд. Это позволяет группировать несколько команд (например, включить проектор + включить звук) и вызывать их одной кнопкой.
* **Инициатор (Invoker):** Класс `Button` имеет один "слот" для команды. Это позволяет менять поведение кнопки "на лету", просто подставляя в слот разные объекты команд.
* **Логика в командах:** В данном примере конкретные команды (например, `PrinterCommand`) инкапсулируют не один вызов метода, а целый сценарий (включить -> напечатать -> выключить). Получатель (`Printer`) ничего не знает об этом сценарии.
* **Управление памятью:** В функции `main()` все устройства и команды создаются на стеке. Это исключает утечки памяти без использования умных указателей и делает код проще для понимания.

## Плюсы
* Полностью отвязывает инициатора (кнопку) от получателя (принтер).
* Позволяет легко создавать новые команды без изменения существующих классов.
* Позволяет реализовать сложные сценарии (Макрокоманды), очереди задач и логирование.
* Идеально подходит для реализации функционала "Отмены" (Undo) и "Повтора" (Redo) — для этого в команду нужно просто добавить метод `undo()`.

## Минусы
* Увеличивает количество классов в системе (отдельный класс для каждой операции).
* Избыточен для простых действий без нужды в очередях, отмене или макросах.
